## Этап 1 — Очистка данных

### 🔴 Deals: аномальный SLA (время ответа менеджера)
**Находка:** SLA max = 26 908 464 сек ≈ **311 дней**. Кто-то ждал ответа менеджера почти год. Среднее SLA сильно искажено такими выбросами — при анализе использовать медиану.

**Решение:** SLA приведён к секундам (int). Значения не трогаем — выбросы анализируем отдельно.

**На слайде:** гистограмма SLA с длинным хвостом — наглядный пример почему mean ≠ типичное значение.

---

### 🔴 Deals: дата закрытия раньше даты создания
**Находка:** 44 сделки где Closing Date < Created Time — логически невозможно (сделка закрылась до того, как создана). Скорее всего баг выгрузки CRM.

**Решение:** Значения не меняем. Добавлен флаг `closing_date_error`. При анализе временных рядов эти строки фильтруем отдельно.

**На слайде:** пример того, что данным из CRM нельзя доверять слепо — нужна проверка.

---

### 🔴 Deals: первый платёж больше суммы предложения
**Находка:** 58 строк где Initial Amount Paid > Offer Total Amount — часть не может быть больше целого. Ошибка ручного ввода менеджеров.

**Решение:** Значения не меняем. Добавлен флаг `amount_error`. При расчёте выручки и среднего чека — фильтруем.

---

### 🟡 Deals: City — 88% пропусков
**Находка:** Из ~21 500 сделок город указан только у ~2 400 (остальные NaN или прочерк). Географический анализ будет крайне ограничен.

**Решение:** Прочерки `-` заменены на NaN. Анализ по городам делаем только там где данные есть, с оговоркой об ограничении.

**На слайде:** честно обозначить ограничение — это показывает зрелость аналитика.

---

### 🟡 Spend: 917 дубликатов с нулевыми расходами
**Находка:** 917 полных дублей строк. Перед удалением проверено: сумма расходов в дублях = 0 €, показы = 0. Это строки канала Bloggers без кампании.

**Решение:** Дубликаты удалены. Итоговая сумма Spend не изменилась — контрольная проверка пройдена.

**На слайде:** пример правильного подхода — не удалять вслепую, а сначала проверить что теряем.

## Этап 2 — Описательная статистика

### Ключевые находки для слайдов

**IAP — IQR = 0 (нестандартная ситуация):**
Более 50% первых платежей — ровно 1 000 €. Q1 = Q3 = 1 000 €, IQR = 0. Метод IQR в таком случае неинформативен — всё что не равно 1 000 € считается "выбросом". Использовали 5-й и 95-й перцентили вместо IQR.

**OTA — выбросов нет:**
Q1 = 5 000 €, Q3 = 11 000 €, IQR = 6 000 €. Нижняя граница = −4 000 € (ниже нуля), верхняя = 20 000 € (выше реального макс 11 500 €). На боксплоте — усы до min/max, точек-выбросов нет. Широкий IQR = разные продукты с разными ценами.

**Course duration — IQR = 0, данные биmodальные:**
11 мес (стандартный курс) + 6 мес (короткий курс) — две разные продуктовые линейки. 6 месяцев — не выброс, а отдельный продукт.

**SLA — обрезка оси до 72 ч:**
Макс SLA = 7 474 ч (311 дней). Один такой выброс делает гистограмму нечитаемой. Показываем на графике только 94.5% лидов (≤ 72 ч), остальные анализируем отдельно.

## Глоссарий кода — объяснения для защиты

Здесь всё что может спросить преподаватель по коду:

| Конструкция | Что значит |
|---|---|
| `iap.dropna()` | Убирает строки где значение пустое (NaN) |
| `iap[~iap.isin([0, 1, 9])]` | Оставляет всё кроме значений 0, 1, 9. `~` = NOT, `isin` = входит в список |
| `iap[(iap != 0) & (iap != 1) & (iap != 9)]` | То же самое через обычные условия. `&` = И |
| `value_counts(normalize=True) * 100` | Частоты как доли (0–1), умноженные на 100 = проценты |
| `reindex(quality_order)` | Переставляет строки в нужном порядке (A→B→C→D→E), а не по частоте |
| `fillna(0)` | Заменяет NaN на 0 — нужно когда в числителе 0 сделок и pandas ставит NaN вместо 0% |
| `[::-1]` | Разворачивает список задом наперёд. Нужен потому что `barh` рисует первый элемент снизу, последний сверху — то есть перевёрнуто. Данные уже отсортированы по убыванию, `[::-1]` разворачивает их перед передачей в график чтобы наибольшее значение оказалось сверху |
| `.mul(100)` | Умножение — то же что `* 100`. Заменено на `* 100` для читаемости |
| `plt.savefig('path/file.png', dpi=100)` | Сохраняет график в файл. `dpi=100` — качество (точек на дюйм) |
| `deals.loc[условие, 'столбец']` | Выбирает строки по условию И одновременно конкретный столбец |
| `groupby('Col', observed=True)` | `observed=True` — без этого pandas выдаёт FutureWarning. Параметр говорит показывать только реальные категории, а не все возможные |
| `groupby().size()` | Считает количество строк в каждой группе |
| `paid / total * 100` | Конверсия: сколько оплатили из всех лидов, в процентах |
| `sort_values(ascending=False)` | Сортировка по убыванию (от большего к меньшему) |
| `q3 + 1.5 * iqr` | Верхняя граница выбросов по методу IQR. Всё выше — статистический выброс |
| `q1 - 1.5 * iqr` | Нижняя граница выбросов по методу IQR. Всё ниже — выброс |
| `dropna=False` в value_counts | Считать NaN как отдельную категорию, не игнорировать |
| `ax.bar_label(bars, fmt='%.0f', padding=4)` | Подписывает значения прямо на столбцах графика. `bars` — объект который вернул `ax.bar()` или `ax.barh()`. `fmt='%.0f'` — формат числа (целое без дробной части). `padding=4` — отступ от края столбца |
| `CTR * 100` | CTR (Click-Through Rate) = Clicks / Impressions. Умножаем на 100 чтобы получить проценты. Например 0.012 → 1.2% |
